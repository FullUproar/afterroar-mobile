# Capacitor setup — Android Studio, signing, Play Store, future iOS

Living doc. Sections fill in as each phase lands. Marked clearly which sections are speculative vs operational.

## Per-app structure (each app is its own Capacitor project)

```
apps/<appname>/
├── src/                       # React app (Vite)
├── public/                    # Static assets
├── index.html
├── package.json
├── vite.config.ts
├── tsconfig.json
├── capacitor.config.ts        # Capacitor's app config (id, name, plugins, native settings)
├── android/                   # Generated by `npx cap add android`. Mostly gitignored — see below.
└── ios/                       # Created at R3. Mostly gitignored — see below.
```

## Capacitor app ID convention

Use reverse-DNS rooted at `me.afterroar` (because the apps are Afterroar-platform clients):

- `apps/register` → `me.afterroar.register`
- `apps/garmr` → `me.afterroar.garmr`
- `apps/passport` → `me.afterroar.passport`

These IDs are permanent — they're the Play Store package name and iOS bundle ID. Choose carefully, never change.

## What to commit vs gitignore in `android/`

Capacitor generates `android/` as a real Android Studio project. Most of it should be gitignored — it's regeneratable from `npx cap sync android`. **Do** commit:

- `android/app/src/main/AndroidManifest.xml` — if we customize permissions (camera, NFC, etc.) beyond Capacitor defaults
- `android/app/src/main/res/` — custom icons, splash screens, strings
- `android/variables.gradle` — version codes, SDK targets
- `android/gradle.properties` — non-secret build config

**Do not** commit:

- `android/build/`, `android/app/build/` — build artifacts
- `android/app/release/*.keystore` — signing keys (NEVER commit, see below)
- `android/local.properties` — points at local Android SDK install
- `android/.gradle/` — Gradle cache

The standard Capacitor `.gitignore` template covers most of this; verify when scaffolding each app.

## Android signing — keystore management

**Critical**: signing keys are generated once per app and **must never change** for the lifetime of the Play Store listing. Lose the keystore = can't push updates. Treat them like database backups.

Plan:

1. Generate keystore for each app on first release: `keytool -genkey -v -keystore <app>-release.keystore -alias <app> -keyalg RSA -keysize 4096 -validity 10000`
2. Store the keystore in **two places**: a 1Password vault entry per app + a backup on encrypted external storage. Never commit to git.
3. Pass keystore path + password to GitHub Actions via repo secrets: `ANDROID_KEYSTORE_BASE64`, `ANDROID_KEYSTORE_PASSWORD`, `ANDROID_KEY_ALIAS`, `ANDROID_KEY_PASSWORD`.
4. The GitHub Actions workflow decodes the base64 keystore at build time, signs the AAB, uploads to Play Store via fastlane.

A separate, single-page doc documenting how to recover from a lost laptop should live at `docs/keystore-recovery.md` once we have the first keystore. That doc will reference 1Password vault names, the encrypted-external-storage location, and the gh-secrets that need to match.

## Play Store — first-time listing checklist (R1)

When R1 is shippable:

1. **Google Play Console account** — $25 one-time fee, 1–2 day identity verification.
2. **App listing**: name, short description, long description, category (Business → Productivity).
3. **Screenshots**: at least 2, max 8. Phone + tablet variants ideally. Take from real Android Studio emulator or connected device.
4. **Privacy policy**: required by Google. Lives at `https://www.afterroar.store/privacy` (or wherever the existing privacy policy is). Listing references this URL.
5. **Internal testing track**: invite 5–10 testers (Shawn + early FLGS owners) with a Google Group email list. Test for 1–2 weeks before promoting to production track.
6. **Production track release**: signed AAB upload, release notes, rollout %. Start at 5–10% staged rollout.

Status: not yet started. R1 needs to actually ship before this matters.

## GitHub Actions — Android build workflow (R1 deliverable)

The CI workflow should:

1. Trigger on tag push matching `register-v*` (per-app tag prefix to allow independent release cadence per app).
2. Use `actions/checkout@v4` + `actions/setup-node@v4` + `actions/setup-java@v4` (JDK 17).
3. Install dependencies: `npm ci`.
4. Build the React app: `cd apps/register && npm run build`.
5. Sync Capacitor: `npx cap sync android`.
6. Decode keystore from secrets, place in `android/app/`.
7. Build signed AAB: `cd android && ./gradlew bundleRelease`.
8. Upload AAB as workflow artifact (downloadable for manual Play Store upload, or fastlane for auto-upload).

Workflow file lives at `.github/workflows/register-android.yml`. To be written when R1 is ready to ship.

## Future iOS — what changes (R3 deliverable)

This section is speculative until R3. Reference [`mobile-strategy.md`](mobile-strategy.md) for the order of operations.

When R3 starts:

1. **Apple Developer Program enrollment** — $99/yr, 1–3 day verification. Start this **first** because it's not gated on code.
2. **Add iOS to the Capacitor project**: `cd apps/register && npx cap add ios`. Generates `ios/` directory.
3. **Build setup**:
   - Either: Mac mini purchase, install Xcode, configure local development.
   - Or: Codemagic free tier, configure `codemagic.yaml` for iOS builds, push and let Codemagic build remotely.
4. **iOS signing** is a different beast from Android — uses Apple-managed certificates + provisioning profiles. Apple now offers "automatic signing" which is actually decent; recommend that for v1.
5. **TestFlight** is the iOS equivalent of Play Store internal testing. Set up testers, run for 1–2 weeks.
6. **App Store Connect listing** — same shape as Play Store but with Apple's review process layered on top. First-time apps can wait 1–3 weeks (or longer) for first approval. Subsequent updates are usually 24–48 hours.

## Plugin reference

|---|---|---|

| `@capacitor/barcode-scanner` (or community) | register | Barcode lookup at register |
| `@capacitor-community/nfc` | passport (later) | Tap-to-check-in at Connect stores |

## Dev loop reference

```bash
# Web-only iteration (fastest)
cd apps/register
npm run dev            # Vite dev server in browser

# Native Android iteration
npm run build          # Build React app
npx cap sync android   # Copy build into Capacitor project
npx cap run android    # Build + install on connected Android device
                       # (requires Android Studio + device in dev mode)

# Open Android Studio for native debugging
npx cap open android

# Future iOS (R3+)
npx cap sync ios
npx cap run ios        # macOS only
npx cap open ios       # Opens Xcode
```
